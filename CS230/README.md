CS 230 – Software Design Portfolio Artifact
Draw It or Lose It – The Gaming Room
Client Summary and Software Requirements

The client, The Gaming Room, requested assistance in expanding their existing Android-based game, Draw It or Lose It, into a scalable web-based application. The goal was to design a distributed system capable of supporting multiple teams and players across various platforms, including Windows, macOS, Linux, Android, and iOS.

The application required unique game and team names, centralized game state management, secure communication, and support for future scalability. The solution needed to operate reliably in a distributed environment while maintaining performance, security, and data integrity.

Strengths in Developing This Documentation

One area I performed particularly well in was clearly structuring and organizing the design document. Each section builds logically from system requirements to architecture, platform evaluation, and final recommendations. I also provided detailed justifications for architectural decisions rather than simply listing technologies.

Additionally, I incorporated object-oriented design principles, distributed system concepts, memory management techniques, and security considerations to demonstrate a well-rounded understanding of system architecture.

Value of the Design Process

Working through the software design document before implementation was extremely helpful. It forced me to think critically about scalability, concurrency, operating system architecture, storage solutions, and distributed communication before writing code.

The design process created a structured blueprint for development, reducing ambiguity and preventing costly redesigns later. It reinforced the importance of planning system architecture early in the development lifecycle.

Areas for Revision and Improvement

If I were to revise one part of this document, I would expand the visual system architecture diagrams and provide additional modeling of database structure and API interactions. While the written explanations are thorough, incorporating more visual documentation would improve clarity for stakeholders and development teams.

Interpreting and Implementing User Needs

I interpreted The Gaming Room’s needs by focusing on scalability, cross-platform accessibility, and centralized state management. Because the application must support multiple concurrent users, I designed a client–server distributed architecture using a centralized service layer.

Considering user needs is critical in software design because technical decisions directly impact usability, reliability, and long-term growth. A system that does not align with user expectations or business goals will fail regardless of technical sophistication.

Approach to Software Design

My approach to designing this application included:

Identifying both functional and non-functional requirements

Evaluating operating system architectures

Applying object-oriented design principles

Considering memory and storage management strategies

Planning for distributed networking and security

In future projects, I will continue using structured design documentation, architectural evaluation, and stakeholder validation before development begins. This method ensures clarity, maintainability, and scalability in complex software systems.

This project strengthened my ability to analyze system architecture, evaluate operating platforms, and communicate technical solutions effectively. It demonstrates my ability to think beyond code and design software systems strategically.
